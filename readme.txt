Каталог ConturTestTask содержит решение MSVS 2013 для заданного тестового задания.

Состав решения:

../readme.txt - данный файл помощи
../runme.bat - стартовый скрипт для быстрого запуска программы
../readme.lnk - ссылка на быстрый запуск документации к исходному коду

./Debug/ - содержит исполняемый файл с отладочной информацией
./Release/ - содержит исполняемый файл без отладочной информации
./ConturTestTask/ - содержит исходные и заголовочные файлы проекта
./Documentation/ - содержит документацию к исходному коду проекта в формате HTML (сгенерирована с помощью системы документирования Doxygen)

Ответы на вопросы в задании:
1. 
Задача: Разработать формат указанных выше файлов.

Решение:
Для решения поставленной задачи использован формат файлов CSV.
Файл с именами должен содержать имена разделенные разделителем ';' либо символом новой строки.
Файл с симпатиями должен содержать строки имен, каждая строка должна начинаться с имени источника симпатии, 
все имена в строке должны быть разделены разделителем ';'

2. 
Задача: 
Разработать программу, которая принимает на вход два указанных выше файла и по требованию выдает один из следующих списков:
	· список всех не любимчиков, то есть детей которые никому не симпатичны
	· список всех несчастных детей, то есть тех, которые не симпатичны ни одному ребенку из тех, кто симпатичен им самим. За исключением тех детей, которым никто не симпатичен.	
	· список любимчиков, то есть всех детей, которые симпатичны максимальному количеству других детей.

Решение:
Состав решения описан в начале данного файла.

3. Задача:
Описать положительные и отрицательные стороны  предложенного решения.

Решение:
Положительные стороны:
- использование формата CSV для файлов исходных данных позволяет осуществлять редактирование исходных данных удобными средствами, такими как, например, MS Excel;
- в качестве внутренней структуры данных (СД) для хранения информации о взаимоотношения между детьми была использована СД Граф; данная СД дает широкие возможности 
к относительно простому получению разного рода косвенной информации, связанной с отношениями между детьми;
- в качестве способа доступа к элементам СД Граф и к его вершинам были реализованы итераторы доступа, которые также облегчают доступ к внутренним данным СД Граф;
- в качестве способа интерактивного взаимодействия пользователя с программой использован подход текстового меню, данный подход дешев в плане реализации и достаточно информативен для пользователя;

Отрицательные стороны:
- текстовое меню не так привлекательно для пользователя как графический пользовательский интерфейс, в следующей версии программы желательно выполнить взаимодействие пользователя
с программой через GUI;
- формат файлов исходных данных CSV является открытым форматом для хранения исходных данных, поэтому любой файл исходных данных может быть модифицирован любым пользователем программы, в 
случае если такой доступ должен быть исключен для посторонних лиц, то необходимо будет дополнить возможности базы данных RelationshipsDatabase возможностью сохранения и загрузки данных в
бинарный формат на жестком диске или даже не удаленном сервере;
- можно было бы сократить издержки на написание программы, взяв в качестве СД не Граф, а таблицу, т.к. она наиболее гармонично подходит для данного вида информации, но в таком случае гибкость
возможностей обработки данных бы уменьшилась.
